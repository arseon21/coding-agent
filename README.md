Итоговый отчет по проекту «Coding Agent SDLC Pipeline»
I. Общее описание решения и архитектура
Цель: разработать автоматизированную агентную систему, имитирующую полный цикл разработки ПО (SDLC) внутри GitHub с использованием LLM-модели (YandexGPT).
Ключевые достижения:
Создана модульная архитектура с разделением ролей на Code Agent (разработчик) и AI Reviewer Agent(ревьюер).
Реализован однократный сквозной процесс SDLC: Issue → Код → PR → Ревью.
Вся работа автоматизирована с использованием GitHub Actions и Docker.
Стек технологий:
Язык: Python 3.11+
LLM: YandexGPT (через API)
GitHub/Git: PyGithub (API) и GitPython (локальные операции)
Инфраструктура: Docker, GitHub Actions.

II. Основной сценарий работы (Демонстрация на видео)
Демонстрация запуска системы от Issue до первого ревью:
1.	Пользователь создает новое Issue с описанием задачи. Триггер: issues: opened. Запускается Job: code-agent.
2.	Code Agent считывает Issue, генерирует код, создает ветку feature/issue-X.
3.	Code Agent создает Pull Request. Триггер: pull_request: opened. Запускается Job: reviewer-agent.
4.	Reviewer Agent получает Diff PR, Issue, отправляет их в YandexGPT.
5.	Reviewer Agent публикует автоматический комментарий в PR (Code Review, Summary).
6.	Финальный PR содержит проверенное решение или требования для исправления.

III. Достигнутые результаты и реализация
Полная автоматизация первого прохода: процесс от создания задачи до первого ревью автоматизирован через GitHub Actions
Контекстное кодирование: code Agent успешно использует LLM для генерации логики, опираясь на содержание Issue и существующую структуру проекта.
Автоматизированный Code Review: AI Reviewer Agent выполняет глубокий анализ, сопоставляя изменения кода (Diff) с исходными требованиями Issue
Универсальность запуска: единый main.py диспетчер, позволяющий запускать агентов как в локальном CLI, так и внутри контейнера GitHub Actions.
Контейнеризация: проект упакован в Docker для обеспечения воспроизводимости и простоты развертывания.
План дальнейшего развития
Полная автоматизация петли: интегрировать в Reviewer Agent логику, которая при вердикте REQUEST_CHANGES автоматически проставляет специальный тег или запускает следующий Job, исключающий ручной триггер
Базовая проверка безопасности: добавить в Reviewer Agent специализированный промпт для поиска распространенных уязвимостей (SQL Injection, XSS) в генерируемом коде.
Тестирование: внедрить автоматический запуск тестов (pytest) в Reviewer Agent Job (как это предусмотрено в YAML), чтобы анализ LLM основывался не только на коде, но и на результатах CI.
